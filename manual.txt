1. SYSTEM ARCHITECTURE (DUAL-CORE)

Unlike standard controllers, this system runs on two separate processor cores to ensure safety:
Core 1 (Control Loop): Dedicated to real-time tasks. It monitors sensors (Ultrasonic, Voltage, Flow) every 1–4 seconds and controls the Motor/Solenoid. It is immune to WiFi lag.
Core 0 (Network Loop): Handles WiFi, Captive Portal, Web Server, MQTT TLS Security, and OTA Updates. If the internet hangs, Core 1 continues to protect the motor.

2. ADVANCED SENSING LOGIC

2.1 Ultrasonic Level Monitoring
Filtering: Uses a 20-sample median filter to ignore "splash" or "ripples."
Range: Configurable for tanks between 1.0 ft and 7.0 ft.
Blind Zone: Includes a hardcoded 10-inch "Buffer Height" to account for sensor placement above the water line.
Error Handling: If the sensor fails (disconnected or obscured) for 10 consecutive cycles, the system enters SENSOR_ERROR and kills the motor.

2.2 ZMPT101B Voltage Intelligence
High-Speed Sampling: RMS voltage is monitored constantly.
Stability Filter: Includes a "Slew Rate Limit" of 3V per sample and a 5-sample median filter. This prevents the pump from tripping during harmless micro-spikes.
Hysteresis (Resume Gap): If the voltage trips, it must return to the safe range by a user-defined gap (e.g., 5V) for 15 seconds before the motor is allowed to restart.

2.3 Flow & Dry-Run Detection (Pin 18)
Initial Grace Period: 60-second bypass upon starting to allow the pipes to prime.
Slug Logic: A 60-second "Persistence Window." If water flow flickers for a few seconds (air bubbles), the system does not trip immediately, preventing "nuisance" alarms.

3. DETAILED OPERATIONAL MODES

3.1 Water Pump Mode
Logic: Starts at Low %, Stops at Full %.
DND Integration: Respects the "Do Not Disturb" schedule unless a manual start is triggered.

3.2 Air Compressor Mode
Pre-Start Valve (Pre-Venting): When the motor needs to start, it first opens the Solenoid (Pin 10) for 5–15 seconds to release head pressure, then starts the motor.
Post-Stop Valve (Post-Venting): After the motor stops, the solenoid opens again to ensure the compressor head is empty for the next cycle.

3.3 Master/Slave Pairing (Multi-Tank)
Role 1 (Master/Sump): Reports status to smartpump/<ID>/status. It includes a "Settling Time" (0–30 mins) where it stays "Busy" after pumping to let silt settle.
Role 2 (Slave/Roof): Subscribes to the Master's topic. It will block pumping if the Master is: Offline, Pumping, Empty, or Settling.

5. PROTECTION SUITES

Thermal Protection: After 60 minutes of continuous runtime, the system forces a cool-down period (5–15 mins).
Voltage Guard: Cut-offs at user-defined High (up to 260V) and Low (down to 150V) levels.
Smart Scheduling (DND): Uses NTP time (GMT+6.5 for Myanmar) to silence the motor during night hours (e.g., 22:00 to 06:00).
Dry-Run Auto-Retry: If the tank runs dry, the system can automatically try again after 30 or 60 minutes.

6. USER INTERFACE & CONTROLS

6.1 Physical Button (Pin 9)
1x Click: Toggle Pump OR Stop Alarm.
Note: The system will block a "Manual Stop" if the water is below the Low Threshold to prevent the tank from staying empty.
2x Click: LCD Info Overlay. Displays:
License: Days remaining.
IP Address: Local network address.
Device ID: Unique Cloud identifier.
Hold 3s: Activates AP Mode (WiFi: Auto-Pump-Config, Pass: 12345678).

6.2 20x4 LCD Display
Big Numbers: The water level (%) and Voltage (V) are displayed using custom-generated large characters across 2 rows for high visibility.
Status Column (Right Side): Shows real-time status: FLOW OK, WAIT SUMP, SNR ERR, OVER V, etc.

7. CLOUD & MAINTENANCE

MQTT Cloud: Secure TLS connection to hivemq.cloud on Port 8883.
Event Logging: Stores the last ~200 events in events.txt (LittleFS). Logs include timestamps for starts, stops, and errors.
License Management: Premium features are tied to a hardware-encrypted license. Expiry disables Cloud control and Auto-Start functions.
OTA (Over-The-Air): Check for updates via the Web Dashboard. The system compares the local version against the GitHub version.txt.
