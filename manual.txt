🏷️ AMK Smart Pump System V2.1 Premium | Full Technical Specification

1. Core Hardware & System Architecture
Microcontroller: ESP32-S3 Dual-Core.
Core 1 (Real-Time Control): Handles logic, sensors, and safety (1s tick).
Core 0 (Networking): Handles WiFi, Web, MQTT, and OTA.
Watchdog (WDT): 30,000ms (30s) hardware timer on both cores.
Pinout Configuration:
GPIO 1/2: I2C SDA/SCL (LCD 20x4, Address 0x27).
GPIO 4: ZMPT101B Voltage Sensor (ADC_11db, 12-bit Resolution).
GPIO 5/6: Ultrasonic Trig/Echo.
GPIO 7: Buzzer (Active HIGH).
GPIO 8: Primary Motor Relay.
GPIO 9: Manual Multi-Function Button (Internal Pull-up).
GPIO 10: Solenoid Valve (Unloader).
GPIO 18: Flow Sensor (Internal Pull-up).
GPIO 48: Onboard NeoPixel RGB LED.

2. Advanced Sensor Intelligence
A. Voltage Guard (ZMPT101B Pipeline)
Sensitivity: 500.0f.
Filtering:
Median of 5 samples.
Outlier Hold: Jumps >18V must persist for 3 cycles (1.5s) before being accepted.
Slew Rate Limit: Maximum 3.0V change allowed per 500ms.
EMA Smoothing: 0.25 * current + 0.75 * previous.
Thresholds: High (230-260V), Low (150-190V), Resume Gap (1-10V).
Wait Logic: 15-second mandatory stabilization timer after power normalization.
B. Ultrasonic Depth Profiling
Sampling: Median of 20 samples (60ms intervals).
Range: 12 inches (Min) to 84 inches (Max).
Buffer (Blind Zone): Top 10 inches ignored to protect the sensor.
Reliability: "SNR ERR" triggered after 10 consecutive invalid readings.

3. Operation Profiles & Logic
A. Operation Modes
Water Pump: Starts at Low % (20-70), stops at Full % (80-100).
Compressor: Solenoid Venting (Pre-start and Post-stop) for 5-15 seconds.
B. Device Roles (Master/Slave)
Standalone: Local control only.
Master (Sump): Reports pump status and source stability.
Slave (Roof): Subscribes to Master's ID. Blocks if Master is pumping or unsafe.
Link Timeout: 5-minute (300,000ms) safety fallback if Master goes offline.
Settling Delay: Master can enforce a 5, 10, 15, 20, or 30-minute "Settling" window after pumping.

4. Protection & Safety Mechanisms
Flow Persistence (Dry-Run):
60s Initial Grace (startup).
60s Slug Window (ignores temporary air pockets).
Logic: If Motor is ON and Flow is missing >60-180s (Configurable) -> Alarm.
Dry-Run Recovery:
Phase 1: DRY_RUN_ALARM (Buzzer oscillate 60s).
Phase 2: DRY_RUN_LOCKED.
Phase 3: Auto-Retry after 30 or 60 minutes.
Thermal Guard: 1-hour continuous run triggers mandatory 5-15 minute rest.
Auto-Resume: Remembers if the pump was running before a Voltage or Thermal interruption and restarts when safe (if not in DND).

5. Connectivity & UI
A. Manual Button Logic (Pin 9)
15ms Debounce: High-speed filter.
Short Click: Toggles Motor. (Decision delayed by 850ms to check for double-click).
Double Click: Shows IP, Device ID, and License status on LCD for 10s.
Long Press (3s): Forces Access Point (AP) mode (SSID: Auto-Pump-Config, Pass: 12345678).
B. Network & Security
MQTT Cloud: HiveMQ TLS (Port 8883) with ISRG Root X1 Cert.
Topics: smartpump/<id>/status, smartpump/<id>/set, and Broadcast smartpump/all/set.
License System: MD5 Signature: Timestamp|Days|ACER123|DeviceID.
Captive Portal: Redirects all DNS (Port 53) to 192.168.4.1 in AP mode.
DND Scheduler: NTP-based time sync with 0.5-step GMT offsets.
C. Visual/Audio Status
Buzzer Patterns: Oscillating (Dry-Run), Pulsed (Sensor Error).
NeoPixel Colors: Red (Error), Blue (Cooling), Purple (Settling), Yellow (Pumping), Green (Online), Cyan (WiFi), Pulsing Blue (Offline).
LCD Big Numbers: Custom 8-character generation for high-visibility Level/Voltage.

6. Maintenance & Logging
Smarter Event Log: events.txt in LittleFS.
Timestamp: [dd/mm HH:MM] format.
Limit: 8,000 bytes (~200 entries).
Rollover: Automatically clears and logs [SYS] Log Rolled Over when full.
OTA: Supports GitHub remote updates and local .bin file uploads via Web UI.
Factory Access: Advanced MQTT settings locked via Key AMK_ADMIN_2026.
