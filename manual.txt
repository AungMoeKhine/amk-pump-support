Briefing Document: AMK Smart Pump and Compressor Control System (v2.1)

Executive Summary

The AMK Smart Pump and Compressor Control System v2.1 is an advanced industrial-grade automation solution built on the ESP32-S3 dual-core architecture. Utilizing the FreeRTOS framework, the system decouples safety-critical motor control from network services to ensure real-time reliability. Key capabilities include dual-mode operation (Water Pump or Air Compressor), a sophisticated Master-Slave configuration for multi-tank environments, and a robust suite of protections including voltage guards, dry-run prevention, and thermal cool-down cycles. Connectivity is managed through a TLS-secured MQTT cloud interface and a local web dashboard, supported by a token-based licensing system and Over-the-Air (OTA) update functionality.

1. System Architecture and Core Logic

Dual-Core Framework (FreeRTOS)

The system leverages the ESP32-S3's dual cores to prevent network latency from interfering with physical hardware safety:

* Core 1 (Control Loop): Dedicated to real-time safety and control logic.
* Core 0 (Network Loop): Manages connectivity services (WiFi, MQTT, Web Server).
* Scheduler Performance: General logic runs on a ~1-second cadence, with ultrasonic measurement cycles every 4 seconds.

Multi-Role Connectivity

The system can be configured in three primary roles:

1. Standalone: Operates independently based on local sensor data.
2. Master (Sump Tank): Publishes pump and status states to an MQTT topic.
3. Slave (Upper Tank): Subscribes to the Master's status to enforce sequential logic.
  * Link Timeout: If the Master does not update for 5 minutes, the Slave enters a fallback behavior.

Sensing Specifications

* Ultrasonic Range: Optimized for heights between 12 and 84 inches.
* Voltage Guarding: Includes high/low cutoffs with a "Resume Gap" and delayed restart to protect against power instability.
* Flow Persistence: Uses "initial grace" and "slug windowing" (60 seconds) to prevent false dry-run alarms during air pockets or initial start-up.

3. Protection and Recovery Mechanisms

The system prioritizes hardware longevity through several automated guards:

* Dry-Run Guard: If the motor is active but no flow is detected within a configurable delay, the system triggers an alarm and locks the motor.
* Auto-Retry: Following a dry-run lock, the system re-attempts operation after a rest period (default 30/60 minutes).
* Thermal Cool-Down: A mandatory rest window is enforced after 1 hour of continuous runtime to reduce motor heat stress.
* Voltage Stabilization: Employs an anti-spike filter and median sampling. If voltage is abnormal, the motor is cut, and a "Wait Seconds" timer must expire before resuming.

4. Connectivity and User Interface

Local and Remote Access

* Web Dashboard: Accessible via local IP or Access Point mode. Provides live data, threshold settings, and maintenance logs.
* Captive Portal: Automatically redirects users to the configuration page when connected to the device's Access Point.
* MQTT Cloud Control: Allows remote command and status monitoring using TLS (Root CA-based) for secure transport.

User Interaction Logic

The manual button supports sophisticated multi-input logic:

* Single Click: Toggles the pump (Start/Stop).
* Double Click: Toggles an information overlay on the LCD (IP address/Device ID).
* Long Press (3s): Manually triggers Access Point (AP) mode for configuration.
* Smart Scheduling (DND): NTP-based time windows prevent auto-starting during night hours (Do Not Disturb).

5. Maintenance and Security

Data Persistence and Security

* NVS (Non-Volatile Storage): Saves all thresholds, WiFi credentials, and runtime states to ensure recovery after power loss.
* Licensing System: A token-based validity system using Base64/MD5 signatures. The system tracks "Valid Days" and disables motor operation upon expiry.
* Event Logging: The system maintains a rolling log (events.txt) in LittleFS, tracking power cycles, sensor errors, and manual overrides.

Firmware Management

* OTA Updates: Supports local firmware uploads and remote update checks via GitHub.
* Diagnostic Strings: The system provides real-time status updates such as FLOW_CHECKING!, SETTLING_WATER!, and VOLT_DELAY.
